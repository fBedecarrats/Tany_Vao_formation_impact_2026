8  Méthodes d’appariement

Cliquer ici pour télécharger la présentation

L’appariement est un des deux centres de gravité du format 5 jours. Son objectif n’est pas de recréer une randomisation parfaite, mais de rendre traités et contrôles plus comparables à partir d’informations observées.

8.1 Pourquoi cette méthode intervient ici

Les chapitres précédents ont montré deux limites :

  • les comparaisons brutes entre groupes sont vulnérables au biais de sélection ;
  • l’avant-après ne suffit pas, car beaucoup d’autres changements peuvent intervenir dans le temps.

L’appariement cherche donc à construire un groupe de comparaison plus proche des unités traitées sur quelques covariables jugées importantes.

8.2 Préparer les données au niveau des aires protégées

On part de l’objet enrichi par mapme.biodiversity, puis on construit un petit jeu de variables d’appariement.

Code 
library(sf)
library(tidyverse)
library(lubridate)
library(mapme.biodiversity)
library(MatchIt)
library(cobalt)

AP_Vahatra <- readRDS("data/AP_Vahatra_mapme.rds") %>%
    portfolio_wide()

Vahatra_defor_noNA <- AP_Vahatra %>%
    mutate(
        surface_m2 = as.numeric(st_area(.)),
        surface_ha = surface_m2 / 10000,
        surface_km2 = surface_m2 / 1000000,
        couv_foret_2000 = `treecover_area_2000-01-01_treecover_ha` / surface_ha * 100,
        altitude = `elevation_2000-02-01_elevation_mean_m`,
        pente = `slope_2000-02-01_slope_mean_degrees`,
        dist_ville = `traveltime_2015-01-01_5k_110mio_traveltime_mean_minutes`,
        traitement = ifelse(year(date_creation) < 2015, 1, 0),
        taux_annuel_deforestation_2000_2014 =
            -((`treecover_area_2014-01-01_treecover_ha` /
                     `treecover_area_2000-01-01_treecover_ha`)^(1/14) - 1) * 100
    ) %>%
    filter(!is.na(couv_foret_2000), !is.na(dist_ville), !is.na(altitude), !is.na(pente))

8.3 Le score de propension

On commence par spécifier une probabilité d’être traité à partir de plusieurs covariables observées.

Code 
pscore_formule <- traitement ~ surface_km2 + couv_foret_2000 + altitude + pente + dist_ville

Cette formule ne doit pas être lue comme une vérité substantielle sur la sélection. Elle sert à résumer la comparabilité observable entre groupes.

8.4 Lancer un appariement simple

Code 
appariement_ap <- matchit(
    formula = pscore_formule,
    family = binomial(link = "logit"),
    method = "nearest",
    discard = "both",
    replace = FALSE,
    distance = "glm",
    data = Vahatra_defor_noNA
)

appariement_ap

Le résultat résume notamment :

  • le type d’appariement ;
  • le nombre d’observations gardées ;
  • les observations écartées ;
  • la cible de l’estimation.

8.5 Les diagnostics sont le point central

L’appariement ne vaut que par les diagnostics qui suivent. C’est le bloc qu’il ne faut pas sacrifier dans le format 5 jours.

8.6 Estimer un résultat pondéré

Une fois l’appariement réalisé, on peut extraire l’échantillon apparié et réestimer le résultat d’intérêt avec les poids produits par l’algorithme.

Code 
donnees_appariees <- match.data(appariement_ap)

broom::tidy(
    lm(
        taux_annuel_deforestation_2000_2014 ~ traitement + surface_km2 +
            couv_foret_2000 + altitude + pente + dist_ville,
        data = donnees_appariees,
        weights = weights
    )
)

8.7 Passage aux mailles

La logique est ensuite transposée à un niveau plus fin, celui des mailles spatiales. Dans le format 5 jours, cette étape repose sur un objet déjà apparié, afin de concentrer le temps de séance sur l’interprétation.

Code 
grille_matched <- readRDS("data/grille_matched.rds")

grille_matched %>%
    st_drop_geometry() %>%
    count(position_ap)
# A tibble: 3 × 2
  position_ap     n
  <chr>       <int>
1 Extérieur    4325
2 Frontière    1146
3 Intérieur    7199
Code 
grille_matched %>%
    st_drop_geometry() %>%
    summarise(
        n = n(),
        poids_moyen = mean(weights),
        distance_moyenne = mean(distance)
    )
# A tibble: 1 × 3
      n poids_moyen distance_moyenne
  <int>       <dbl>            <dbl>
1 12670           1            0.234

Cette transition prépare directement la double différence sur mailles.

8.8 Ce qu’il faut retenir

L’appariement n’est pas seulement une procédure automatique. C’est une méthode de construction d’un groupe de comparaison qui oblige à justifier les covariables, à examiner le support commun et à lire sérieusement les diagnostics d’équilibre.
